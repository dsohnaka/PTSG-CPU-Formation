# Layer 4 — Verification Evidence / 検証エビデンス

The evidence that the specification and the silicon agree. **CC0.** Format and rules follow [PTSG-Core Layer 4](https://github.com/dsohnaka/PTSG-Core/tree/main/04_Verification_Evidence): VCD as the primary format, mandatory `observation.md` verdicts, a living conformance matrix, negative results as first-class evidence, predictions frozen before demonstration. / 仕様とシリコンの一致の証拠。**CC0。** 形式と規則は[PTSG-Core Layer 4](https://github.com/dsohnaka/PTSG-Core/tree/main/04_Verification_Evidence)に従う: VCD一次形式、必須observation.md、生きた適合マトリクス、一級エビデンスとしての負の結果、実演前の予言凍結。

**This is also the only place in this repository where resource figures may appear** — as measurements pinned to a named revision, device, toolchain version, and synthesis configuration (*Measured, not promised*; see the root README). / **本リポジトリでリソース数値が現れてよい唯一の場所でもある**——命名された改訂・デバイス・ツールチェーン版・合成設定に釘付けされた実測として（*約束せず、測って刻む*;ルートREADME参照）。

Planned verification platform: Terasic C5G (pure Cyclone V GX fabric, no HPS) — the absence of a hard processor is the methodological control that makes the fabric-resident control claim testable without confound. / 検証予定プラットフォーム: Terasic C5G（純Cyclone V GXファブリック、HPSなし）——ハードプロセッサの不在は、ファブリック常駐制御の主張を交絡なしに検証可能にする方法論的対照である。
